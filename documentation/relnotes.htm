1. OVERVIEW (top of page)

AddonSoftware™ is a cost-effective, modular, full-featured and fully integrated enterprise resource planning solution for such mid-market industries as Accounting, Distributing, and Manufacturering.  

2. WHAT IS NEW IN THIS RELEASE (top of page)

Version 8.01 takes advantage of the technological advancements provided by Barista™ and BBj® to deliver an innovative software package know for its structural quality. Together with Barista, Version 8.01 offers a leap forward in the efficiency of software customization and the creation of vertical applications. AddonSoftware continues its 25-year history of constant improvement - additional features, increased functionality, and updated technology - in this release..

  • Built using the Barista Application Framework
  • Graphical User Interface
  • Navigate with either keyboard or mouse
  • Normalized data structure
  • Xkeyed files eliminate extra cross-references
  • Utilizes Barista's robust Inquiry subsystem to view, sort, filter and export records
  • Application and topic help
  • Multi-platform and multi-language
  • Enhanced program code structure
  • Built-in email and fax capability
  • Barista's DocOut subsystem provides print preview as well as multiple output/file save formats

3. KNOWN ISSUES (top of page)

  • The Barista Application Framework requires BBj 8.0. Earlier versions of BBj may produce unexpected results
  • When running under Linux, the KDE desktop environment must be used.

4. SUPPORT (top of page)

If you purchased your AddonSoftware product from a distributor, reseller, or OEM, please contact them for technical support. They will answer your question or contact us on your behalf.

Technical support for version 8.01 is available via e-mail at addon-developer@addonsoftware.com.

When submitting a technical support question, please provide the following information:
     1. The product serial number.
     2. The name of the product and the product's version number.
     3. A complete description of the problem, including the product error and the TCB(10) value, if available.
     4. A screen shot of the problem.

Other pertinent information includes:
     1. The operating system name, JRE, level and version.
     2. Is the error consistently reproducible, or is it sporadic?
     3. Is the error isolated to a specific machine or user?
     4. Is this a new installation or has the system been in place and operating normally?
     5. Is the problem network related/specific?
